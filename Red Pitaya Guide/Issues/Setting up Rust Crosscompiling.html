Setting up rust crosscompiling

What is Rust

Rust is a rather new systems programming language which was designed to be able to compile and run efficient code and that by beeing memorysafe. This means, that writing code in rust is much more prone against writing code which is unsafe in a Memory access sense. This makes develeping in rust much more stable and less prone to errors then programming in other lowlevel langugages for example C.
Appart from the fact that rust provides a reliable and efficeient way for memory safe programming it commes with a lot of other tools which ease you live as a systems programmer. For example rust includes a whole dependency management system named Cargo, which helps you to incorporatre third party libraries on the fly, without the need of a compilcated build system. Furthermore you can easily search for aviable liblrarys called crates on https://crates.io/
A more detailed desription and and a lot of additional information on how to get startted you can find on:
https://www.rust-lang.org/

Starter guide Book: https://doc.rust-lang.org/book/

Install Rust on your host machine

A detailed explanation on how to install rust on your windows machine is provided here:
https://doc.rust-lang.org/book/ch01-01-installation.html

Setup a new rustproject with cargo

Starting a new rustproject is suuuper easy! Just navigate to the folder, where you want to start a new project, open a command promt and enter: 

cargo new (APPLICATION NAME)

and e value, you created a new rustproject.

Set up tools for crosscompilation

Crosscompiling in rust is farely as well easy compared to other low level programming languages

Since the whole project setup is manged by your cargo enviroment you just need to set the correct build tools and linkers and the rest is managed by cargo.

To now set the build tools you need to run serveral commands in a promt which you open up in your project.

The first command calls the rust setup manager (rustup) to add the correct build tools for the target enviroment. In case of the Redpitaya it is a 32bit ARM-Cortex processor running linux. Therefore you need to run:

```cmd command rustup target add arm-unknown-linux-gnueabihf 
</div>



Since Rust does not come with a standalone linker the next thing you must do is to downlead linker tool which can be used by Rust. You can find one here:[https://releases.linaro.org/components/toolchain/binaries/latest-5/arm-linux-gnueabihf/](/not_created.html)   

There you download the mingw32_arm-linux-genueabihf.tat.xz   
You anzip this file to some arbitrary directory.    

The you need to go to your rustproject again and open up a new folder named .cargo   
in this folder you create a new file called config.toml   

Open this file and enter:   
<div class="lang-rust">
```rust
[build]
target = "arm-unknown-linux-gnueabihf"

[target.arm-unknown-linux-gnueabihf]
linker = "C:/your_linker_path/gcc-linaro-5.5.0-2017.10-i686-mingw32_arm-linux-gnueabihf/bin/arm-linux-gnueabihf-gcc.exe"

where you replace the path in the linker variable with your own one.

Now you just need to go back to the command promt again and run: 

cargo build --release

or: 

cargo build --debug

depending wether you want to use a debug or a relase build.

If the compileprocess exits successfully you can navigate to the targetfolder in your project folder where a new path named /arm-linux-gnueabihf should be generated. Navigater into this path and copy the Binaryfile there to the filesystem of the RedPitaya. Then you should be able to run it.

Cograts you ran your first Rustprogramm on the RedPitaya!!!!!!!!!

Red pitaya reserved ddr memory patch

Explanation

To enable safe direct memory access into the external memory, it is highly recommended to set up the system so there is a reserved memory region to which Linux system doesn't write and thus something doesn't break down when DMA dumps loads of data.

This is done by modifying u-boot script (Linux configuration at startup) and device tree (specifying "peripherals") on the Red Pitaya. There are four files that are important:

  • devicetree.dts - source file for the device tree
  • devicetree.dtb - compiled device tree
  • u-boot.script - source file for U-boot script
  • u-boot.scr - compiled U-boot script
Caution

While editing files on Windows, make sure that the source files have end-of-line characters set as LF (UNIX standard) and not CR+LF (windows standard)!

Applying the patch

  1. Insert SD card (the one with Red Pitaya ecosystem) to SD card reader
  2. Copy devicetree.dtb and u-boot.scr to root folder
    1. These files are located at GITREPO/ddr_patch or created following Recompiling the patch files
    2. Default configuration is split in half, leaving 256MB memory for Linux and the rest for DMA
  3. Reinsert SD card into the Red Pitaya, power it up and connect
  4. Run head /proc/meminfo in terminal on the RP and you should see something like this if everything went well and you used the default files: 
MemTotal:         213976 kB
MemFree:           89400 kB
MemAvailable:     160388 kB
Note

Make sure to change the address range for the DMA (S_AXI_HP0) in Vivado.

For default config that means
- Master Base Address 0x1000_0000
- Master High Address 0x1FFF_FFFF
- Range 256M

Recompiling the patch files

Modifying original files

  1. Open devicetree.dts
  2. Find line (line 683 by default) 
memory {
        device_type = "memory";
        reg = <0x0 0x20000000>;
    };
  1. Change second reg value to desired upper limit for the Linux operation. By default: 
memory {
        device_type = "memory";
        reg = <0x0 0x10000000>;
    };
  1. Make sure to leave EOL as LF, save and close file
  2. Open u-boot.script
  3. Find block (line 10 by default) 
if test ${hw_rev} = 'STEM_125-10_v1.0'
then  
  setenv high     0x10000000
  setenv bootargs ${bootargs} mem=256M
else 
  setenv high     0x20000000
fi
  1. Change the second setenv high to the exact same value as second reg value in devicetree.dts. By default: 
if test ${hw_rev} = 'STEM_125-10_v1.0'
then  
  setenv high     0x10000000
  setenv bootargs ${bootargs} mem=256M
else 
  setenv high     0x10000000
fi
  1. find line setenv bootargs and append mem=YOURMEMSIZE
    1. By default it's (...) rootwait uio_pdrv_genirq.of_id="generic-uio" mem=256M
  2. Again make sure EOL is set to LF, save and close

Recompiling modified files

  1. You can use either Red Pitaya itself, or another Linux machine, procedure should be the same, this will follow instructions for Red Pitaya
  2. Connect to RP and copy u-boot.script and devicetree.dts to root
  3. Open terminal and run: 
mkimage -A arm -T script -C none -d ./u-boot.script ./u-boot.scr

dtc -O dtb -o devicetree.dtb devicetree.dts
  1. If everything ran succesfully, you should now have two new files: u-boot.scr and devicetree.dtb
  2. Follow Applying the patch on instructions how to apply the new files

Finding clock-capable pin

Look at the schematic and the pins has to have a suffix SRCC (Single-region clock capable) or MRCC (Multi-region clock capable)

If specific package pin is clock capable can be checked via .tcl command: 

get_property IS_CLK_CAPABLE [get_package_pins {H16}]
Note

Note that when using only one of the pins of the differential input, clock has to be connected to the P pin of the pair. [UG865 p.16]

For Red Pitaya, clock-capable pins should be the following (according to the Z7020 Red Pitaya schematic)

U14 SRCC_34 1V8 Daisy connector 1 P
U15 SRCC_34 1V8 Daisy connector 1 N
N18 MRCC_34 1V8 Daisy connector 3 P
P19 MRCC_34 1V8 Daisy connector 3 N
L16 SRCC_35 3V3 DIO_5_P
L17 SRCC_35 3V3 DIO_5_N
K17 MRCC_35 3V3 DIO_3_P

MRCC vs SRCC (click to expand)

The clock capable pins in a 7 series FPGA serve two purposes; access to the local clocking resources and access to the global clocking resources.

If you are using the global clocking resources (BUFG, BUFH, MMCM, PLL) then the MRCC and SRCC have exactly the same capability - there is no difference between the two.

If you are using the local clocking resources (BUFR and BUFIO), then then the SRCC and MRCC can both only drive only the BUFIO and BUFR located in the same clock region. The BUFIO can then only drive the IOB flip-flops and high speed clock of the ISERDES in the same I/O bank and the BUFR can clock all the logic (except the high speed clock of the ISERDES) in the same clock region.

The only difference between the SRCC and MRCC is that the MRCC can also drive the BUFMR. The BUFMR can then drive the BUFIO/BUFR in the same clock region as well as in the clock regions above and below the MRCC. This would generally be used for "ChipSync" (source synchronous) interfaces that need to use more pins than are available in one I/O bank.

That being said, if you don't need the extra functionality of the BUFMR, then it is probably best to use the SRCC pins first.

Implementing external clock in Vivado

Relevant resources:

  • UG472 (Clocking resources user guide)
  • UG899 (I/O and clock planning)
  • DS187 (DC/AC switching characteristics data sheet)

Steps:

1) Choose one of the pins above and note it's package number
2) Open your Vivado project
3) Create new port and specify it's name
1) For single-ended create just one (e.g. ext_clk_i)
2) For differential clock, create two ports (e.g. ext_clk_i_p and ext_clk_i_n)
4) Open red_pitaya.xdc constraint file (Sources tab -> Constraints -> constrs_1)
5) Find relevant package pin number among already defined pins and delete/comment out existing port definition
6) This should be minimum required code to define a new 1.8V single-ended clock: 

# Pin package pin assigment and I/O standard
set_property PACKAGE_PIN L16 [get_ports {ext_clk_i}]
set_property IOSTANDARD LVCMOS18 [get_ports {ext_clk_i}] #set to 1.8V logic

# Create 1 MHz clock
create_clock -period 1000.0 -name ext_clk [get_ports ext_clk_i]

7) This should be minimum required code to define a new 1.8V differential clock: 

# Pin package pin assigment and I/O standard
set_property PACKAGE_PIN L16 [get_ports {ext_clk_i_p}]
set_property PACKAGE_PIN L17 [get_ports {ext_clk_i_n}]
set_property IOSTANDARD DIFF_HSTL_I_18 [get_ports {ext_clk_i_p}]
set_property IOSTANDARD DIFF_HSTL_I_18 [get_ports {ext_clk_i_n}]

# Create 1 MHz clock
create_clock -period 1000.0 -name ext_clk [get_ports ext_clk_i_p]
